## ヒアリング結果サマリー

### 確認できた事項

- SSG実現方式: vite-react-ssg
- シンタックスハイライト: Shiki（ビルド時、クライアントJS不要）
- 検索: MiniSearch（ビルド時インデックス生成 + クライアント検索、bigram日本語対応）
- 画像欠落=ビルドエラー / 同一日はフォルダ名降順 / Lighthouse Performance 90+

### 設計方針の決定事項

- 完全静的構成（実行時DB・APIなし）のため database-schema.sql / api-endpoints.md は作成しない
- マークダウン処理はunifiedエコシステム（remark/rehype）+ gray-matter + Zodバリデーション
- 状態管理はReact Context + カスタムフックのみ（Redux等は不採用）
- テーマはTailwind `class`戦略 + head内インラインスクリプトでFOUC防止

### 残課題

- サイト名・サイト説明文（SiteConfig.title / description）の文言 → 実装フェーズで決定
- Shikiのテーマ選定（ライト/ダーク用のカラーテーマ名） → 実装フェーズで決定
