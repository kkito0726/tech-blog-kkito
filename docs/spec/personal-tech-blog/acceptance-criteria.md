## REQ-001/002: マークダウン記事管理とfrontmatter 🔵

**信頼性**: 🔵 *ユーザー依頼・ヒアリングQ6より*

### Given（前提条件）
- リポジトリに記事ディレクトリが存在する

### When（実行条件）
- frontmatter付きマークダウンファイルを追加してビルドを実行する

### Then（期待結果）
- 記事がパースされ、title / date / description / draft が正しく読み取られる
- 記事詳細ページ用のデータが生成される

### テストケース

#### 正常系

- [ ] **TC-001-01**: 必須項目のみのfrontmatter 🔵
  - **入力**: `title` と `date` のみを持つ記事ファイル
  - **期待結果**: ビルド成功。記事ページ・一覧に反映される
  - **信頼性**: 🔵 *ヒアリングQ6より*

- [ ] **TC-001-02**: 全項目を持つfrontmatter 🔵
  - **入力**: `title` / `date` / `description` / `draft: false` を持つ記事ファイル
  - **期待結果**: ビルド成功。descriptionが一覧・メタタグに使用される
  - **信頼性**: 🔵 *ヒアリングQ6より*

#### 異常系

- [ ] **TC-001-E01**: 必須項目欠落 🟡
  - **入力**: `title` が無い記事ファイル
  - **期待結果**: ビルドが対象ファイル名を含むエラーメッセージで失敗する（REQ-104）
  - **信頼性**: 🟡 *グローバルルール（入力バリデーション）から推測*

- [ ] **TC-001-E02**: YAML構文エラー 🟡
  - **入力**: frontmatterのYAMLが不正な記事ファイル
  - **期待結果**: ビルドが対象ファイル名を含むエラーメッセージで失敗する（EDGE-001）
  - **信頼性**: 🟡 *EDGE-001から推測*

- [ ] **TC-001-E03**: スラッグ重複 🔵
  - **入力**: 日付プレフィックス除去後に同一スラッグとなる記事フォルダ2件（例: `2026-01-01-foo/` と `2026-02-01-foo/`）
  - **期待結果**: ビルドが重複フォルダ名を含むエラーで失敗する（EDGE-002）
  - **信頼性**: 🔵 *追加ヒアリングQ9より*

## REQ-011/012: 画像コロケーションとURL形式 🔵

**信頼性**: 🔵 *追加ヒアリングQ8・Q9より*

- `content/posts/YYYY-MM-DD-<スラッグ>/` フォルダに `index.md` と画像ファイルが存在する

### When（実行条件）
- 本番ビルドを実行し、記事詳細ページにアクセスする

### Then（期待結果）
- 記事URLが `/posts/<スラッグ>/`（日付なし）で公開される
- 相対パス参照の画像がハッシュ付きアセットURLに解決され、表示される

### テストケース

#### 正常系

- [ ] **TC-011-01**: 相対パス画像の解決 🔵

  - **期待結果**: 生成HTMLで画像がハッシュ付きURLに解決され、ページ上に表示される
  - **信頼性**: 🔵 *追加ヒアリングQ8より*

- [ ] **TC-012-01**: URLからの日付除去 🔵
  - **入力**: フォルダ名 `2026-07-14-vite-ssg-setup/` の記事
  - **期待結果**: `/posts/vite-ssg-setup/` でアクセスできる
  - **信頼性**: 🔵 *追加ヒアリングQ9より*

#### 異常系

- [ ] **TC-011-E01**: 参照先画像の欠落 🔵
  - **入力**: 存在しない画像への相対パス参照を含む記事
  - **期待結果**: ビルドが対象ファイル名を含むエラーで失敗する
  - **信頼性**: 🔵 *設計ヒアリングQ5で確定*

### テストケース

- [ ] **TC-003-01**: 降順ソート 🟡

  - **期待結果**: 新しい順に表示される
  - **信頼性**: 🟡 *一般仕様から推測*

#### 境界値

  - **入力**: 公開記事が0件の状態
  - **期待結果**: 空状態メッセージが表示され、ビルドは成功する（EDGE-101）
  - **信頼性**: 🟡 *EDGE-101より*

- [ ] **TC-003-B02**: 同一公開日 🔵
  - **入力**: 同じ `date` を持つ記事2件
  - **期待結果**: フォルダ名の降順で安定して表示される（EDGE-104）
  - **信頼性**: 🔵 *設計ヒアリングQ5で確定*
