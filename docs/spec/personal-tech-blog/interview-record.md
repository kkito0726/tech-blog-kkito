# 個人テックブログ ヒアリング記録

**作成日**: 2026-07-14
**ヒアリング実施**: 要件定義フェーズ（新規プロジェクトのため、既存資料なし・全項目をユーザーヒアリングで確定）

## ヒアリング目的

新規プロジェクトであり既存のPRD・設計文書・実装が存在しないため、ユーザー依頼文で不明確だった点（ホスティング先、レンダリング方式、機能スコープ、記事メタデータ、リリース方針）を明確化するためのヒアリングを実施した。

## 質問と回答

### Q1: この要件の作業規模について教えてください

**質問日時**: 2026-07-14
**カテゴリ**: 優先順位
**背景**: 要件定義書の詳細度（EARS要件・ユーザーストーリー・受け入れ基準の粒度）を決定するため。

**信頼性への影響**:
- 詳細なユーザーストーリー・受け入れ基準・非機能要件・エッジケースを含む完全な要件定義書を作成することが確定。

### Q2: ブログのデプロイ先（ホスティング）はどこを想定していますか？

**質問日時**: 2026-07-14
**カテゴリ**: 未定義部分詳細化
**背景**: 依頼文では「GitHub Actionsで自動反映」とあるがデプロイ先が未指定。デプロイ先によりワークフロー構成・base設定が変わるため。

**回答**: GitHub Pages（推奨）

**信頼性への影響**:
- REQ-403（GitHub Pagesホスティング）を 🔵 で確定。
- REQ-405（サブパス配信対応）を派生要件として追加（🟡）。

### Q3: レンダリング方式はどちらを希望しますか？（SEOに影響します）

**質問日時**: 2026-07-14
**カテゴリ**: 未定義部分詳細化
**背景**: Vite + ReactはデフォルトでSPAとなり、OGP・SEOメタタグがクローラーに認識されない。SSGの要否はアーキテクチャの根幹に関わるため。

**回答**: 静的生成（SSG）（推奨）

**信頼性への影響**:
- REQ-008（ビルド時静的生成）を 🔵 で確定。
- REQ-007（OGP・SEOメタタグ）の実現方式が確実になった。

### Q4: ブログに必要な機能を選択してください（複数選択可）

**質問日時**: 2026-07-14
**カテゴリ**: 追加要件
**背景**: 依頼文にサイト機能の記載がないため、スコープを確定する。選択肢: タグ・カテゴリ分類 / 記事検索 / RSSフィード / ダークモード。

**回答**: ダークモード、記事検索（タグ・カテゴリ分類とRSSフィードは非選択）

**信頼性への影響**:
- REQ-009（記事検索）・REQ-010（ダークモード）を 🔵 で追加。
- タグ・カテゴリ、RSSをWon't Have（REQ-406）として 🔵 で確定。

### Q7: リリーススコープについて教えてください

**質問日時**: 2026-07-14
**カテゴリ**: 優先順位
**背景**: 機能の実装順序・フェーズ分割を決定するため。

**回答**: 段階的リリース（推奨）

**信頼性への影響**:

### Q8: 記事を保持するフォルダの追加ルールはどうする想定か？（画像添付のケースを含む）【追加ヒアリング】

**質問日時**: 2026-07-14（要件定義書作成後の追加確認・ユーザー発の質問）
**カテゴリ**: 未定義部分詳細化
**背景**: 初回要件定義では記事ディレクトリの具体的な構成（特に画像の置き場所）が未定義だった。「1記事1フォルダ方式（コロケーション）」と「フラット方式 + public/images/」の2案を提示した。

**回答**: 1記事1フォルダ方式（コロケーション）を採用。記事フォルダは `content/posts/YYYY-MM-DD-<スラッグ>/`、本文は `index.md`、画像は同フォルダ内に置き相対パスで参照する。

**信頼性への影響**:
- REQ-001 を具体化し 🔵 を維持
- 新規要件 REQ-011（画像コロケーションと相対パス解決）を追加（信頼性レベル: 🔵）

---

### Q9: 記事URLに日付を含めるか？【追加ヒアリング】

**質問日時**: 2026-07-14
**カテゴリ**: 未定義部分詳細化
**背景**: フォルダ名を `YYYY-MM-DD-スラッグ` に統一したため、URLに日付部分を含めるかどうかの確定が必要だった。

**回答**: 含めない。URLは `/posts/<スラッグ>/`（公開日を変更してもURLが変わらない。スラッグ重複はビルドエラーで検出）。

**信頼性への影響**:
- 新規要件 REQ-012（URL形式）を追加（信頼性レベル: 🔵）
- EDGE-002（スラッグ重複時のビルドエラー）が 🟡 → 🔵 に向上

## ヒアリング結果サマリー

### 確認できた事項

- デプロイ先はGitHub Pages、レンダリングはSSG
- 機能スコープ: 記事一覧・詳細、シンタックスハイライト、TOC、OGP・SEO、検索、ダークモード
- 非スコープ（Won't Have）: タグ・カテゴリ、RSS、コメント機能
- frontmatter: title / date（必須）、description / draft（任意）
- 段階的リリース（Phase 1: コア、Phase 2: 検索・ダークモード）
- 記事ディレクトリは1記事1フォルダ方式（`content/posts/YYYY-MM-DD-<スラッグ>/index.md` + 同フォルダに画像）
- 記事URLは `/posts/<スラッグ>/`（日付を含めない）

### 追加/変更要件

- REQ-009（記事検索）、REQ-010（ダークモード）、REQ-102（draft制御）を依頼文に対して追加
- 追加ヒアリング（Q8・Q9）により REQ-011（画像コロケーション）、REQ-012（URL形式）を追加

### 残課題

- Lighthouseスコア等の具体的なパフォーマンス目標値（NFR-003 🔴）
- 同一公開日の記事の並び順の第2ソートキー（EDGE-104 🔴）
- リポジトリ名・公開URL（base設定値）の確定
- 検索ライブラリ・SSG手法の選定（設計フェーズで決定）
